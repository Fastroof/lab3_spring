10.	В чому полягає різниця між сервлетами та JSP-сторінками?
> Кожен із них має як переваги так і недоліки, є різні таблиці в Інтернеті. <br>Servlet - код Java, написання сторінок тяжче ніж з JSP, він відіграє роль контролера в MVC, він швидший за JSP, він може прийняти всі протокольні запити, керування сеансом потрібно вмикати вручну, вся логіка повинна бути в єдиному файлі, модифікація дуже складна, він не має вбудованих неявних об’єктів, він немає методу запуску JavaScript на стороні клієнта.<br> JSP - код на основі html, легко створити сторінку. JSP — це вид у підході MVC для показу результату. JSP повільніший за Servlet, тому що першим кроком у життєвому циклі hasJSP є переклад JSP у код Java, а потім компіляція. JSP приймає лише HTTP-запити. У JSP керування сеансами вмикається автоматично. У JSP бізнес-логіка відокремлена від логіки представлення за допомогою JavaBeans на стороні клієнта. JSP змінюється швидко, просто потрібно натиснути кнопку оновлення. У JSP є вбудовані неявні об’єкти. Під час запуску JavaScript на стороні клієнта в JSP використовується перевірка на стороні клієнта.<br> Отже JSP однозначно кращий вибір для нашого проекту.
11.	Як виглядає життєвий цикл JSP-сторінки? Що таке трансляція та компіляція JSP-сторінок?  
> JSP life cycle: 
> - Translation (Переклад або Трансляція) JSP в сервлет<br>Це перший крок життєвого циклу JSP. Ця фаза перекладу стосується синтаксичної правильності JSP. Тут файл .jsp -> .java.
> - Compilation (Компіляція) JSP сторінки <br>.java -> .class
> - Classloading (Завантаження класу) <br> Клас сервлета, який було завантажено з джерела JSP, тепер завантажується в контейнер. (У Spring контейнер сервлетів створюється автоматично і для всіх)
> - Instantiation (Примірник) <br> Тут генерується екземпляр класу. Контейнер керує одним або кількома примірниками, надаючи відповіді на запити.
> - Initialization (Ініціалізація) <br> Метод jspInit() викликається лише один раз протягом життєвого циклу відразу після створення екземпляра Servlet з JSP.
> - Request processing (Обробка запиту) <br> Обслуговуються виклики JSP. Приймаються об’єкти запиту та відповіді як параметри.
> - JSP Cleanup (Очищення JSP) <br> Кінець життя
13.	В чому полягає різниця між директивою <%@ include%> та елементом <jsp:include>?
> 1. Найважливіша відмінність між <%@ include%> і <jsp:include> полягає в тому, що <%@ include%> обробляється під час трансляції, а <jsp:include> обробляється під час запиту, тобто коли запит надходить на обробку.
> 2. Друга важлива відмінність між <%@ include%> та <jsp:include> полягає в тому, що <%@ include%> може використовувати як відносний, так і абсолютний шляхи, а <jsp:include> завжди використовує відносний шлях.
> 3. Третя ключова відмінність полягає в тому, що <%@ include%> може включати лише вміст ресурсу, наприклад HTML або CSS, але не оброблятиме динамічний ресурс, а <jsp:include> обробляє динамічний ресурс, і результат буде додано до виклику JSP.
> 4. У <%@ include%> ви не можете передати будь-який інший параметр, наприклад запит, відповідь або будь-яку конфігурацію, але за допомогою <jsp:include> ми можемо передати інший параметр також за допомогою <jsp:param > тега.
> 5. П’ята відмінність є розширенням попередньої відмінності, тобто за допомогою <%@ include%> ви не можете передати об’єкт запиту чи відповіді виклику jsp до включеного файлу чи JSP або навпаки, але за допомогою <jsp:include> можна передати об'єкт запит/відповідь.
> 6. Остання, але не менш важлива різниця між <%@ include%> та <jsp:include> полягає в тому, що перша використовує атрибут файлу для вказівки ресурсу, який буде включено, і тому також називається включення файлу, а <jsp:include> використовує атрибут сторінки для вказівки ресурсу, який потрібно включити, також відома як page include у JSP.
14.	Що таке JavaBean? Як їх використовувати у JSP-сторінках?
> JavaBean - це клас, який відповідає певним критеріям, а саме: всі поля приватні і використовуються тільки гетери та сетери, дефолтний конструктор без аргументі, реалізована серіалізація (імплементує Serializable). JavaBean можна використовувати в JSP за допомогою <jsp:useBean>. Приклад: 
> <pre><code>&lt;jsp:useBean id = "date" class = "java.util.Date" /&gt;
> &lt;p&gt;The date/time is &lt;%= date %&gt;</code></pre>
> 
> Результат: ```The date/time is Thu Sep 30 11:18:11 GST 2010 ``` <br> Також можна використати <jsp:getProperty/> або <jsp:setProperty/> для отриманя або задання значення поля біна. Приклад:
> <pre><code>&lt;jsp:useBean id = "test" class = "com.test.Test" /&gt;
>   &lt;jsp:setProperty name = "test" property = "temp" value = "It's test"/&gt;
> &lt;/jsp:useBean&gt;
> &lt;p&gt;&lt;jsp:getProperty name = "test" property = "temp"/&gt;&lt;/p&gt;</code></pre>
> 
> Результат: ```It's test```
